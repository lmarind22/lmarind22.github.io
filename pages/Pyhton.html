I - Introducción

En el contexto de trabajo colaborativo que incluye la programación de códigos en un lenguaje determinado, es crucial:

  • Establecer un marco común de desarrollo:
    • Lenguaje de programación e infraestructura de desarrollo asociada.
    • Coherencia de la versión del lenguaje y su infraestructura de programación.
    • Control y trazabilidad de las versiones del programa y sus cambios.

En esta guía, dentro del contexto de la elaboración de proyectos colaborativos, se ha optado por utilizar Python. Esta selección se basa en la pertinencia y facilidad de uso, así como en la importancia de su comunidad internacional. Python se ha convertido en el lenguaje de referencia para el tratamiento masivo de datos y es ideal para la creación de prototipos de flujos de análisis rápidos y eficientes.

La infraestructura software asociada será:

  • Python xx.xx
    • Lenguaje de programación
    • Librerías básicas: Numpy, Pandas, Scipy, Matplotlib
    • Librerías específicas según el problema en consideración
  • Jupyter-Lab notebooks
    • Interfaz de enseñanza, documentación, intercambio de código y explotación en webAPP.
  • Spyder como entorno de desarrollo, “debug” y test de los códigos.
  • Git y Github para la administración de versiones y repositorio del código.

Nos concentraremos en esta primera etapa en la gestión de Python en ambientes virtuales. En próximas versiones se documentarán otros aspectos.

II - Instalación de una versión específica de Python

La última versión de Python se puede encontrar en el sitio oficial en la sección de "Download" a fecha de 19/08/2024.

La versión actual de Python en el sitio es Python 3.12. Aunque sea posible descargar la versión 3.13, se recomienda descargar la versión estable 3.12. Recuerda que las versiones anteriores también pueden ser descargadas si otros scripts han sido desarrollados con versiones anteriores, ya que la compatibilidad ascendente en Python está asegurada, pero la descendiente no.

Instalación en Windows 11.x.x

  1. Descarga el ejecutable de instalación de la versión Python 3.12 desde el sitio oficial.
  2. Ejecuta el instalador y elige la opción de instalación personalizada para poder seleccionar los directorios de instalación.
  3. Selecciona el directorio donde se desea instalar python, esta dirección debe ser fácil de reconocer y ubicar:
    4. C:\Users\login\Pyversion\Python312

    En este caso, de antemano, se crea la carpeta PyVersion para guardar las instalaciones de las distintas versiones de python con las que se desean trabajar. Además, se sugiere que en el directorio C:\Users\login\ se cree una carpeta PyVenv, para guardar los entornos virtuales de los proyectos asociados.

    Termina la instalación. Ahora estás listo para crear los entornos virtuales asociados a los diferentes proyectos.

III - Creación del entorno virtual de trabajo para proyectos determinados

Para asegurar la permanencia y compatibilidad en el funcionamiento de los códigos de un proyecto, cada proyecto de Python debe incluir la definición del entorno de trabajo: versión de Python, librerías, etc. Este entorno se presenta en la forma de un directorio que contiene las versiones de los módulos y librerías necesarios.

Procedimiento

  1. Abre una ventana de línea de comando cmd desde el menú de inicio de Windows.
  2. Navega al directorio de la versión de Python que se utilizará con el comando:
    3. CD c:\Users\login\PyVersions\Python312

    login es el nombre del usuario que se está usando desde el ordenador

  3. Desde ese directorio, crea el entorno virtual de trabajo asociado al proyecto con el comando:
    4. python -m venv C:\Users\login\PyVenv\LearningCurve

    Esto creará una nueva carpeta en el directorio C:\Users\login\PyVenv\ llamada LearningCurve, aquí podrá poner el nombre del proyecto que esté trabajando

  4. Navega al directorio del entorno virtual:
    5. CD C:\Users\login\PyVenv\LearningCurve
  5. Activa el entorno virtual con el comando:
    6. C:\Users\login\PyVenv\LearningCurve\Scripts\activate

    Verifica que no hay ningún módulo o librería adicional con el comando: pip freeze.

  6. Para desactivar el entorno virtual, utiliza el comando deactivate.

IV - Instalación de librerías y módulos necesarios

El entorno virtual de cada proyecto tendrá dos componentes: uno propio del proyecto y otro específico al usuario. Estos aspectos se documentan en el archivo requirements.txt

Procedimiento

  1. Abre una ventana de comando, navega al repertorio del entorno virtual y actívalo:
    2. CD C:\Users\login\PyVenv\LearningCurve\Scripts\activate
  2. Instala las librerías necesarias con el comando pip install

    3. Se tiene dos opciones una de estas si se está trabajando sobre un proyecto donde ya se ha definido todo el entorno virtual (por ejemplo, cuando se está trabajando con un clon de Github o se comparte el proyecto en local) en ese caso solo hay que obtener el fichero requirements.txt, se copia en el directorio del proyecto. Luego se instala todo el entorno virtual de trabajo con el comando siguiente:

    pip install -r C:\users\login\PyVenv\LearningCurve\requirements.txt

    Todos los módulos y librerías necesarios para la ejecución del proyecto serán instaladas y el entorno virtual estaría listo.

    El otro escenario es comenzar de cero, para esto, una vez que se ha creado el entorno virtual en directorio específico (III-4) de esta guía. De aquí en adelante, suponemos que estamos en el directorio del entorno virtual. En nuestro ejemplo. C:\Users\cb223735\PyVenv\LearningCurve\ Para verificar que no hay ningún módulo la comando pip freeze debe regresar una línea vacía, o una lista de los módulos/librerías instaladas.

    A partir de este punto se procede a instalar uno por uno los diferentes módulos/librerías. Por ejemplo:

    
        pip install numpy
        pip install scipy
        pip install matplotlib
        pip install jupyterlab
  3. Crea el archivo requirements.txt para documentar las librerías necesarias:
    4. pip freeze > C:\Users\login\PyVenv\LearningCurve\requirements.txt
  4. Para lanzar Jupyter Lab, utiliza el comando jupyter-lab. El programa preguntará por el núcleo que se desea utilizar y aparecerá el entorno virtual creado.

V - Supresión de un entorno virtual

Una vez que el proyecto haya sido terminado y ya no se necesite más, se puede eliminar el entorno virtual:

  1. Desinstala todos los módulos y librerías utilizados con el comando:
    2. pip uninstall -r requirements.txt -y
  2. Desactiva el entorno virtual
    3. deactivate
  3. Borra todos los archivos y directorios asociados.
